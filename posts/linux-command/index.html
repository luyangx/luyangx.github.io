Linux 命令

April 6, 2014

记录几个跟排查性能有关, 还有比较重要的 Linux 命令

vmstat

监控虚拟内存的使用情况, 输出如下(单位:KB)

$ vmstat 2 4 # 每隔 2 秒监控1次, 一共监控 4 次
procs -----------memory---------- ---swap-- -----io---- -system-- ------cpu-----
 r  b   swpd   free   buff  cache   si   so    bi    bo   in   cs us sy id wa st
 1  0      0 203340 221480 2320808    0    0    20    28    1    1  4  5 91  1  0
 0  0      0 203812 221480 2320808    0    0     0     0 2708 4197  6 14 81  0  0
 0  0      0 203812 221484 2320824    0    0     0    28 2855 4395  7 15 78  0  0
 0  0      0 204060 221484 2320824    0    0     0     6 2723 4268  5 14 81  0  0
  • procs: 进程相关
    • r: 运行中和等待运行的的进程, 如果一直大于 CPU 核心数, 系统可能负载很高
    • b: 等待IO的进程数量
  • memory: 内存相关
    • swpd: 切换到内存交换区的内存大小, si, so=0 时, 系统性能还算正常
    • free: 空闲内存大小
    • buff: 用于读写 buff 占用内存大小
    • cache: page cache 占用内存大小
  • swap: 内存交互相关
    • si: 由内存进入内存交换区数量, swap in
    • so: 由内存交换区进入内存数量, swap out
    • bi: 从磁盘读入数据的总量, block in
    • bo: 从磁盘写入数据的总量, block out
  • system: 系统相关
    • in: 每秒 cpu 中断次数
    • cs: 上下文切换, 越小越好, context switch
  • cpu: CPU 相关
    • us: 用户 CPU 时间, 过长性能会有问题, 需要优化用户程序
    • sy: 系统 CPU 时间, 过长性能会有问题
    • id: 空闲 CPU 时间
    • wt: 等待 CPU 时间, 过长性能会有问题, 可能磁盘IO性能低

iostat

监控 I/O 使用情况, 可以方便查看 CPU, 网卡, tty设备, 磁盘, CD-ROM 等设备的负载情况, 单位: KB

$ iostat 1 2 # 每隔 1 秒监控 1 次, 一共监控 2 次
Linux 3.10.0-957.21.3.el7.x86_64 (iz2zednvbin1s0wn630o0tz) 	04/06/2014 	_x86_64_	(2 CPU)

avg-cpu:  %user   %nice %system %iowait  %steal   %idle
           3.55    0.00    4.53    0.66    0.00   91.26

Device:            tps    kB_read/s    kB_wrtn/s    kB_read    kB_wrtn
vda               6.32        41.61        57.11  210574195  289032248

avg-cpu:  %user   %nice %system %iowait  %steal   %idle
           5.86    0.00   14.64    2.09    0.00   77.41

Device:            tps    kB_read/s    kB_wrtn/s    kB_read    kB_wrtn
vda               4.00        28.00       140.00         28        140

第一行打印 Linux 系统版本, 架构和 CPU 个数

  • avg-cpu: CPU 相关
    • %user: 用户使用 CPU 百分比
    • %nice: 用户使用带 nice 的 CPU 百分比
    • %system: 系统使用 CPU 百分比
    • %iowait: 等待 IO 设备 CPU 百分比, 过高, IO设备有性能问题
    • %steal: 虚拟 CPU 等待时间百分比
    • %idle: 空闲 CPU 百分比
  • Device IO设备相关:
    • tps: 设备每秒传输次数, 多个 IO 请求会合并成 1 次 IO 请求
    • kB_read/s: 每秒从设备读取的数据大小
    • kB_wrtn/s: 每秒向设备写入的数据大小
    • kB_read: 读取的总数据大小
    • kB_wrtn: 写入的总数据大小

查看设备使用率(%util)和响应时间(await):

$ iostat -d -x -k 1 2
Device:         rrqm/s   wrqm/s     r/s     w/s    rkB/s    wkB/s avgrq-sz avgqu-sz   await r_await w_await  svctm  %util
vda               0.07     4.83    2.77    3.54    41.61    57.11    31.25     0.05    9.00   10.08    8.16   2.23   1.41

Device:         rrqm/s   wrqm/s     r/s     w/s    rkB/s    wkB/s avgrq-sz avgqu-sz   await r_await w_await  svctm  %util
vda               0.00    29.00    3.00    2.00    24.00   124.00    59.20     0.06   13.00   20.67    1.50  12.80   6.40
  • rrqm/s: 每秒合并读操作次数
  • wrqm/s: 每秒合并的写操作次数
  • r/s: 每秒读次数
  • w/s: 每秒写次数
  • rsec/s: 每秒读扇区数
  • wsec/s: 每秒写扇区数
  • rkB/s: 每秒读 K 字节数
  • wkB/s: 每秒写 K 字节数
  • avgrq-sz: 平均每次操作的数据大小
  • avgqu-sz: 平均队列长度
  • await: 平均每次操作的等待时间(ms)
  • svctm: 平均每次操作的服务时间(ms)
  • %util: 一秒中 I/O 操作百分比

如果 %util 接近 100, I/O 请求太多, I/O系统已经高负荷, 磁盘可能存在性能问题, 正常小余 70%

top

实时监控系统运行状态

top - 20:33:02 up 58 days, 14:08,  1 user,  load average: 0.49, 0.50, 0.56
Tasks: 127 total,   6 running, 121 sleeping,   0 stopped,   0 zombie
%Cpu(s): 10.6 us, 13.1 sy,  0.0 ni, 74.0 id,  2.1 wa,  0.0 hi,  0.1 si,  0.0 st
KiB Mem :  8009180 total,   172828 free,  5302284 used,  2534068 buff/cache
KiB Swap:        0 total,        0 free,        0 used.  2357268 avail Mem

  PID USER      PR  NI    VIRT    RES    SHR S  %CPU %MEM     TIME+ COMMAND
19294 apache    20   0  499304  25512  13228 R   5.3  0.3   0:00.16 php
 3418 root      20   0 2517316  67380   2524 S   4.3  0.8 929:46.71 java
19295 apache    20   0  496848  24484  12556 R   3.7  0.3   0:00.11 php
 2217 jenkins   20   0 3150524 361196   5760 S   3.3  4.5 621:28.05 java
19296 apache    20   0  491956  23028  11808 R   3.0  0.3   0:00.09 php

第一行: 当前时间, 已运行多长时间, 当前登录用户数和系统负载情况

  • load average: 系统负载, 3个值分别是 1m, 5m, 15m 前的平均负载值, 用来判定系统是否负载过高, 如果持续大于 CPU 个数, 负载过高, 需要优化

第二行: 进程相关

  • Tasks: 进程总数
  • running: 正在运行进程数
  • sleeping: 睡眠的进程数
  • stopped: 停止的进程数
  • zombie: 僵尸进程数

第三行: CPU 相关

  • sy: 系统占用 CPU 百分比
  • ni: 用户用 nick 占用 CPU 百分比
  • id: 空闲 CPU 百分比
  • wa: 等待 IO 的 CPU 百分比
  • hi: 硬中断
  • si: 软中断

第四行: 内存相关

  • Mem: 物理内存大小
  • used: 物理内存已用大小
  • free: 空闲内存大小
  • buffers: 系统用于缓存的内存大小

第五行: Swap 交换器相关

  • Swap: 交换区大小
  • used: 交换区已用大小, 大于0, 内存平均, 优化程序或增加内存
  • free: 空闲交换区大小
  • cached: 交换区缓存大小

实际使用内存大小: free + buffers + cached 程序已用内存大小: used - (buffers - cached)

第六行以后: 进程相关

  • PID: 进程id, PPID: 父进程id
  • UID: 进程所属用户ID
  • PR: 优先级
  • NI: nice值, 越小表示高优先级, 反之表示低优先级
  • VIRT: 进程使用虚拟内存大小(KB)
  • RES: 进程使用的, 未被换出的物理内存大小(KB)
  • SHR: 共享内存大小(KB)
  • S: 进程状态, status
    • D: 不可中断睡眠状态, uninterruptible sleep
    • R: 运行, running
    • S: 睡眠, sleeping
    • T: 停止, stopped by job control signal
    • t: 跟踪, stopped by debugger during trace
    • Z: 僵尸进程, zombie
  • %CPU: 进程使用 CPU 百分比
  • %MEM: 进程使用物理内存百分比
  • TIME+: 进程使用 CPU 时间(1/100s)
  • COMMAND: 进程执行的命令

free

查看物理内存使用情况

$ free -h
              total        used        free      shared  buff/cache   available
Mem:           7.6G        5.0G        145M        1.1M        2.4G        2.3G
Swap:            0B          0B          0B
  • Mem: 内存相关
    • total: 总物理内存大小
    • used: 已用内存大小
    • free: 空闲内存大小
    • shared: 共享内存大小
    • buff/cache: 用于系统 buff/cache 内存大小
  • Swap: 交换区相关
    • total: 总大小
    • used: 已用大小
    • free: 空闲大小

df

查看文件系统磁盘空间情况

$ df -h
Filesystem      Size  Used Avail Use% Mounted on
/dev/vda1       197G   56G  133G  30% /
devtmpfs        3.9G     0  3.9G   0% /dev
tmpfs           3.9G     0  3.9G   0% /dev/shm
tmpfs           3.9G  1.1M  3.9G   1% /run

如果某项 Use% = 100%, 需要清理垃圾文件了

其他命令

// 访问日志中访问最多前10个IP
$ awk '{print $1}' /tmp/access.log | uniq -c | sort -hr | head -n 10

// 传送文件到服务器目录
$ scp /tmp/access.log zhs@172.18.0.2:/var/www

// 查看目录大小
$ du -sh /tmp

// 查看系统信息
$ uname -a

// 查看命令位置
$ whereis

// 查看 PHP 相关进程
$ ps aux | grep php

// 列出 TCP 网络连接
$ lsof -i tcp

// 查看某个端口被哪个程序占用
$ netstat -anpt | grep 80

// grep 同时查看前后多少行
$ grep -C 10 php /tmp/access.log
// grep 同时查看后多少行
$ grep -A 10 php /tmp/access.log
// grep 同时查看前多少行
$ grep -B 10 php /tmp/access.log
