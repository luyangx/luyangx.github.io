HTTP/1.1 . 连接/缓存/安全

May 2, 2014

HTTP/1.1 目前在互联网上被广泛使用, 记录一下 HTTP1.1 重要的几块内容, 包括连接, 缓存控制等.

连接 Connection

串行连接

浏览器中每个 HTTP 请求串行的建立连接, 加载速度慢, 连接延时叠加, 如下图:

并行连接

浏览器中每个 HTTP 请求都建立自己的连接, 加载速度快, 连接延时重叠, 如下图:

打开大量连接会消耗很多资源, 浏览器会将并行连接总数限制为一个娇小的值, 一般是4个, chrome 是6个.

持久连接

一般持久连接和并行连接配合使用. 持久连接有2种类型:

HTTP/1.0+ - Keep-Alive

客户端可以通过 Connection: Keep-Alive 来请求建立一条持久连接, 如果服务端响应中没有 Connection: Keep-Alive, 则标识服务端不支持, 连接会被客户端关闭.

现在浏览器默认使用 Keep-Alive

HTTP/1.1 - Persistent

工作机制比 Keep-Alive 更优. HTTP/1.1 持久连接默认是激活的, 除非显式发送 connection: close, 连接才会关闭, 否则连接一直是打开状态.

管道化连接

将度欧条请求放入到队列, 当第一条请求发出时, 后面的请求也可以开始发送了, 可以降低网络延时, 提高性能.

缓存

HTTP GET 报文的基本缓存处理包括7个步骤

  1. 接收 - 缓存从网络中读取抵达的请求报文
  2. 解析 - 缓存对报文进行解析, 提取出 URL 和各个首部
  3. 查询 - 缓存查看是否有本地副本可以, 如果没有, 就获取一份副本保存在本地
  4. 新鲜度检测 - 缓存查看已缓存的副本是否足够新, 如果不是, 就询问服务器是否有更新
  5. 创建响应 - 缓存会用新的首部和已缓存的主体来构建一条响应报文
  6. 发送 - 缓存通过网络将响应发回给客户端
  7. 日志 - 缓存可选地创建一个日志文件来描述这个事务

HTTPS

HTTPS 将 HTTP 协议与一组强大的对称/非对称和基于证书的加密技术结合在一起, 在安全的传输层上发送 HTTP 内容. HTTP 安全层是通过 TLS 来实现的.

SSL 握手

在发送加密的 HTTP 报文前, 需要进行一次 SSL 握手, 完成以下工作:

  • 交换协议版本号
  • 选择一个两端都了解的密码
  • 对两端的身份进行认证
  • 生成临时的会话密钥, 以便加密信道

HTTP 状态码

分类

总体范围 已定义范围 类别
100~199 100~101 信息
200~299 200~206 成功
300~399 300~305 重定向
400~499 400~415 客户端错误
500~599 500~505 服务器错误

状态码

状态码 原因 含义
100 Continue(继续) 收到了请求的起始部分, 客户端应继续请求
101 Switching Protocols(切换协议) 服务器正根据客户端的知识将协议切换成 Upgrade 首部列出的协议
200 OK 服务器已成功处理请求
201 Created(已创建) 对那些要服务器创建对象的请求来说, 资源已创建完毕
202 Accepted(已接收) 请求已接收, 当服务器尚未处理
203 Non-Authoritative Information(非权威信息) 服务器已将事务处理成功, 只是实体首部包含的信息不是来自原始服务器, 而是来自资源的副本
204 No Content(无内容) 响应啊博文包含一些首部和一个状态行, 但不包含实体的主题内容
205 Reset Content(重置内容) 另一个主要用于浏览器的代码. 意思是浏览器应该重置当前页面上所有的 HTML 表单
206 Partial Content(部分内容) 部分请求成功
300 Multiple Choices(多项选择) 客户端请求了实际指向多个资源的 URL. 这个代码时和一个选项列表一起返回的, 然后用户可以选择他希望使用的选项
301 Moved Permanently(永久移除) 请求 URL 已移走, 同时响应中包含一个 Location 首部给出资源现在所在位置
302 Found(已找到) 同 301, 只不过是临时的
303 See Other(参见其他) 告诉客户端应该用另一个 URL 获取资源, 这个 URL 在响应 Location 首部中
304 Not Modified(未修改) 资源没有修改过
305 Use Proxy(使用代理) 通过代理访问资源, 代理位置在 Location 首部中
307 Temporary Redirect(临时重定向) 和 301 类似.
400 Bad Request(坏请求) 告诉客户端发生了异常
401 Unauthorized(未授权) 访问资源前, 先进行身份认证
402 Payment Required(要求付款) 未使用
403 Forbidden(禁止) 服务器拒绝请求
404 Not Found 服务器无法找到请求
405 Method Not Allowed(不允许使用的方法) 不支持请求方法, 响应中会返回允许使用的方法
406 Not Acceptable(无法接受) 服务器没有资源与客户端可以接受的 URL 相匹配
407 Proxy Authentication Required(要求进行代理认证) 同 401
408 Request Timeout(请求超时) 请求超时
409 Conflict(冲突) 发出的请求在资源上造成了冲突
410 Gone(消失了) 同 404
411 Length Required(要求长度指示) 服务器要求在请求报文中含有 Content-Length 首部时使用该代码
412 Precondition Failed(先决条件失败) 服务器无法满足客户端的请求条件
413 Request Entity Too Large(请求实体太大) 实体太大
414 Request URI Too Long(请求 URI 太长) URI 太长
415 Unsupported Media Type(不支持的媒体类型) 媒体类型不支持
416 Request Range Not Satisfiable(请求范围未满足) 请求范围无效或未满足
417 Expectation Failed(无法满足期望) 服务器无法满足 Expect 给出的期望
500 Internal Server Error(内部服务器错误) 服务器遇到错误, 无法提供请求服务
501 Not Implemented(未实现) 服务器无法满足客户端请求的某个功能
502 Bad Gateway(网关故障) 代理服务器或网关遇到来自响应链中上游的无效响应
503 Service Unavailable(服务不可用) 服务器目前无法提供服务
504 Gateway Timeout(网关超时) 网关或代理等待另一台服务器的响应时超时
505 HTTP Version Not Supported(不支持的 HTTP 版本) 服务器不支持或不愿支持的协议

Accept

请求首部, 客户端用 Accept 通知服务器可以接收哪些媒体类型.

举例

Accept: text/*, image/*
Accept: text/*, image/gif, image/jpeg; q=1

Accept-Charset

请求首部, 客户端用 Accept-Charset 通知服务器可以接受哪些字符集或哪些是优选字符集.

举例

Accept-Charset: utf8

Accept-Encoding

请求首部, 客户端用 Accept-Encoding 通知服务器可以接受哪些编码方式.

举例

Accept-Encoding: gzip

Accept-Language

请求首部, 客户端用 Accept-Language 通知服务器可以接受哪些语言或哪些是优选语言.

举例

Accept-Language: zh-cn

Accept-Ranges

响应首部, 用来告知客户端是否请求资源的某个范围. 如果该首部有值, 说明服务器允许对指定资源的哪个范围类型进行访问.

客户端可以在没有收到该首部时, 对某个资源发起范围请求. 如果服务器不支持, 会以适当的状态码响应, 将 Accept-Ranges 置 none.

举例

Accept-Ranges: none
Accept-Ranges: bytes

Age

响应首部, 告知接收端响应已产生多长时间.

HTTP/1.1 缓存必须在发送的每条响应中都包含1个 Age 首部.

举例

Age: 60

Allow

响应首部, 通知客户端可以对特定资源使用哪些 HTTP 方法.

405 响应必须包含 Allow 首部.

举例

Allow: #Method

Authorization

请求首部, 客户端发送, 用来向服务器回应自己的身份信息. 客户端收到 401 响应后, 需在请求中包含该首部.

405 响应必须包含 Allow 首部.

举例

Authorization: Basic 123abc

Cache-Control

通用首部, 用于传输对象的缓存信息.

举例

Cache-Control: no-cache

Connection

通用首部, 该首部的值是一个标记列表, 标记对应各种首部名称.

举例

Connection: close

Content-Encoding

实体首部, 用于说明是否对某对象进行过编码. 客户端根据此信息对报文进行解码.

举例

Content-Encoding: compress, gzip

Content-Language

实体首部, 用于告知对象是哪种语言.

举例

Content-Language: zh-cn, en

Content-Length

实体首部, 实体主体部分的长度或尺度.

举例

Content-Length: 2417

Content-Location

实体首部, 给出报文的实体部分对应的 URL. 客户端重定向.

举例

Content-Location: http://www.baidu.com

Content-Range

实体首部, 请求传输某个范围内的文档, 并给出文档的长度.

举例

Content-Range: bytes 500-999 / 5400

Content-Type

实体首部, 媒体类型

举例

Content-Type: text/html; charset=utf8

Cookie

扩展请求首部, 用于客户端识别和跟踪的扩展首部

举例

Cookie: XXXID=abc123

Date

通用首部, 服务器响应报文的创建日期和时间

举例

Date: Wen, 02 May 2014 10:14:05 GMT

ETag

通用首部, 报文实体标记, 标识资源.

举例

ETag: "11e92a-457b-31345aaa"

Expires

实体首部, 响应失效的日期和时间. 浏览器可以缓存一份副本, 在给定的时间之前, 不需要再询问服务器资源是否有效.

举例

Expires: Wen, 02 May 2014 10:14:05 GMT

If-Modified-Since

请求首部, 客户端用 GET 方法请求服务器上的资源, 在上次请求资源后, 该资源是否有修改过. 如果没有修改, 服务端返回 304 Not Modified 响应, 否则服务器会正常响应请求

举例

If-Modified-Since: Wen, 02 May 2014 10:14:05 GMT

If-Match

请求首部, 客户端发起条件请求, 服务端会对比客户端发过来的实体标记是否与请求资源标记相等, 匹配成功, 返回实体.

举例

If-Match: "abc123"

If-None-Match

请求首部, 客户端发起条件请求, 提供一个实体标记列表, 服务端会一一匹配, 只在都不匹配时, 才会返回资源.

举例

If-None-Match: "abc123-456cde-789fgh" // * | ETag

If-Range

请求首部, 客户端发起条件请求, 需要验证范围内资源是否有效, 无效的话会获取新的资源.

举例

If-Range: "abc123-456cde-789fgh" // HTTP-date | ETag

If-Unmodified-Since

请求首部, 客户端发起条件请求, 服务器查看首部日期值, 只有从该首部提供的日期后, 对象未修改过, 才会返回对象

举例

If-Unmodified-Since: Wen, 02 May 2014 10:14:05 GMT

Last-Modified

实体首部, 提供实体最后一次被修改的相关信息.

举例

Last-Modified: Wen, 02 May 2014 10:14:05 GMT

Location

响应首部, 将客户端导向某个资源的地址.

举例

Location: "http://www.baidu.com"

Pragma

请求首部, 用于随报文传送一些指令. 指令通常用来控制缓存行为

举例

Pragma: no-cache // 等价于 Cache-Control: no-cache

Range

实体首部, 请求某实体的部分内容. 它的值说明了报文所包含实体的范围.

举例

Range: bytes=500-1500

Referer

请求首部, 服务器可以知道客户端从哪里获得其请求的 URL.

举例

Referer: http://www.baidu.com/index.html

Set-Cookie

扩展响应首部, 设置 Cookie

举例

Set-Cookie: uid=123; xxx

Upgrade

通用首部, 为报文发送者提供了一种手段, 使其指定另一种可能完全不同协议并将此意愿向外广播. 服务端发送 101 Switching Protocols 响应时, 必须包含该头部.

举例

Upgrade: HTTP/2.0

User-Agent

请求首部, 应用程序的名称, 可能还会包含其他信息, 例如操作系统等

举例

User-Agent: Mozilla/5.0 (Macintosh; Intel Mac OS X 10_15_4) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/81.0.4044.129 Safari/537.36
